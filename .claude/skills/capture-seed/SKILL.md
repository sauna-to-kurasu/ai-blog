# Blog Seed キャプチャ

現在のセッションの会話内容を振り返り、記事ネタになりうる事象を抽出して `docs/blog-seeds.md` に追記する。

## 手順

1. 現在のセッションで行った作業を振り返る
2. 以下に該当するものを特定:
   - ハマりポイントの解決
   - 便利な設定・ツールの発見
   - 定量データ（ビルド時間、サイズ、スコア等）
   - 非エンジニアに説明が必要な概念の言語化
3. `docs/STRATEGY.md` の記事ロードマップと照合し、関連記事候補を特定
4. `docs/blog-seeds.md` の既存エントリと重複がないか確認
5. 新規ネタを所定フォーマットで末尾に追記

## 出力

追記した内容をユーザーに報告:
- 何件のネタを記録したか
- それぞれどの記事候補に紐づくか
- 「新規候補」の場合はSTRATEGY.mdへの追加を提案
