# コンテキスト管理ルール（PROJECT_CONTEXT / PRIVATE_CONTEXT）

## 2層構成の概要

| ファイル | git管理 | 内容 |
|---|---|---|
| `docs/PROJECT_CONTEXT.md` | あり（公開） | ブログ運営に必要な事実情報（個人特定情報はサニタイズ済み） |
| `docs/PRIVATE_CONTEXT.md` | なし（gitignore） | 個人・家族の詳細、居住地、生活パターン |

**記事執筆時は両方を読むこと。**

## 振り分けの判断基準

> **「この情報単体、または他の公開情報との組み合わせで、個人・住所・勤務先を特定できるか？」**
>
> 迷った場合は PRIVATE_CONTEXT.md に入れる（安全側に倒す）。

## PRIVATE_CONTEXT.md に入れるもの（非公開）

| カテゴリ | 例 |
|---|---|
| 特定可能な個人属性 | 職種の詳細（部署名）、勤務時間、資格の等級 |
| 家族の特定情報 | 具体的な年齢、資格名、健康状態 |
| 居住地の特定情報 | 市区町村名、実家の場所 |
| 行動パターン | 起床・就寝時刻、通勤手段・時間 |
| 職場の実績 | 表彰歴、具体的な金額の成果 |
| 訪問先の具体名 | 旅行先（市区町村レベル） |
| メールアドレス・連絡先 | 個人・法人のメールアドレス |
| 金融情報 | 振込先銀行名、支払い日 |

## PROJECT_CONTEXT.md に入れるもの（公開・サニタイズ済み）

| カテゴリ | 書き方のルール |
|---|---|
| 職業 | 「メーカー勤務」（部署名は省略） |
| 家族構成 | 「妻」「未就学の男の子2人」 |
| 居住地 | 「東海地方」まで（市区町村は書かない） |
| 使用機材 | そのまま記載OK（MacBook Air、Galaxy Z Fold7等） |
| コスト金額 | そのまま記載OK（ブログで公開済みの数値） |
| 開発環境・ツール | そのまま記載OK |

## サニタイズの具体例

| 元の情報 | PROJECT_CONTEXT.md | PRIVATE_CONTEXT.md |
|---|---|---|
| 「○○市に住んでいる」 | 「東海地方在住」 | 具体的な市区町村名 |
| 「妻の実家の○○市で作業した」 | （記載しない） | 具体的な場所 |
| 「子供は○歳と○歳」 | 「未就学の男の子2人」 | 具体的な年齢 |
| 「定時は○時」 | （記載しない） | 具体的な時刻 |

## 新しい情報を追記する際のワークフロー

1. ユーザーから新しい事実情報を受け取る
2. 上記の判断基準で振り分け先を決定
3. 該当ファイルに追記
4. PROJECT_CONTEXT.md に追記する場合、個人特定につながる具体値はサニタイズ
5. サニタイズした場合、元の具体値は PRIVATE_CONTEXT.md にも記載

## PROJECT_CONTEXT.md への情報反映ルール

### トリガー（厳守）

**ユーザーからコミット＆プッシュの依頼があったとき**、以下の順序で実行する:

1. まず PROJECT_CONTEXT.md への反映を行う
2. 反映が完了してからコミット＆プッシュする

### 反映すべき情報の種類

| カテゴリ | 例 |
|:---|:---|
| ブログ運営の事実 | 使用ツール、ワークフロー、コスト実績 |
| 開発環境の変更 | 新しいツール導入、設定変更 |
| AI活用の知見 | Claude Code運用のコツ、プロンプト設計 |
| 費用・コストの新データ | 実績値の更新、新しい請求データ |

### 反映してはいけない情報

- 記事の文章表現・構成（コードから読み取れる）
- 一時的な作業メモ
- 推測・仮説（ユーザーから確認が取れた事実のみ）
