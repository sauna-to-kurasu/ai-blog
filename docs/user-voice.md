### 2026-04-03 / Claude Codeの始め方 — ターミナル・料金・学び方

- **やったこと**: 記事No.3ブラッシュアップの深掘り。ターミナル初体験の感覚、料金プランの決断、これから始める人へのメッセージについて語った
- **感想（ターミナル）**: 「最初は少し怖さがあった。でも設定が終わればキーボードで叩くことは特になくて、ChatGPTのセッションと同じように言葉を伝えるだけ。UIがちょっと黒いだけで、やってることはChatGPTと同じだよねって気づいた。一回経験すれば全然気にならない」
- **感想（料金）**: 「3日で上限に達したのはCursorの話。Claude CodeはProプランで最初は問題なかったが、やりたいことが溢れてきて上限に。Proプラン時代にすでに$100くらい課金していたので、Maxの方が絶対お得だと。奥さんに『お小遣いの範囲内でやるから』と伝えたら『それならいいんじゃない』と。お小遣いはかなり減ったが満足。Maxだと普通に使って上限の20%、ガッツリやっても50-60%。Agent Teamsも使って全然上限にいかない」
- **感想（伝えたいこと）**: 「まず触ってみるが一番重要。調べなくてもいい。エラーが出てもスクショ撮ってClaude Codeに見せればいい。重要なのはコーチングで自分の強み・得意・価値観を見極めること。そしてニッチを攻めて一次情報を蓄えてClaude Codeでアウトプットする」
- **感想（新しい学び方）**: 「今までインプット→アウトプットがセオリーだったが、これからは先にアウトプットが出てきて、わからないところを後から聞く。これが新しい学び方」
- **感想（ステップアップ）**: 「ターミナルが苦手ならデスクトップアプリから入ればいい。ChatGPT感覚でやれる。自分も最初そちらから入った。Agent TeamsやAutoモードを使いたくなったタイミングでターミナルに来ればいい。ターミナルの環境構築もデスクトップアプリのClaude Codeに聞けば作れる」
- **温度感**: 非常に高い。読者への強いメッセージ性あり
- **関連記事候補**: No.3ブラッシュアップ（反映済み）、講座セクション

### 2026-04-02 / WordPressからAI+Astroへの体験変化

- **やったこと**: 2026年1月にWordPressでブログを始めたが、途中からClaude Code + Astro + Cloudflareに移行
- **感想**: 「昔、WordPressでブログを作っていたときは、本当にすごく時間がかかっていた。記事の構成を考えたり、考え終わったらそれをマウスでぽちぽちWordPressでデザインを整えたりとか。ただ、それも結局は自分がいいなと思っているだけで、本当のプロの目線ではどうかというところが欠けていた。今は自分がいいなというのは選びつつも、プロ目線での提案の中から選べていく。そういったところがすごく魅力。これはもうWordPressでは無理」
- **温度感**: 非常に高い。WordPress時代との比較から来る実感
- **核心的洞察**: AIの価値は「自動化」だけでなく「プロ品質の選択肢を提示してくれること」。非エンジニアにとって、自分のセンスだけに頼らずプロレベルの提案から選べる体験が最大の差別化
- **関連記事候補**: 講座の訴求全体、No.1体験記の深掘り版、Phase 2「AIに聞けば済む時代に、なぜ僕はブログを書くのか」

### 2026-04-03 / ブログ遍歴の全体像（3-4年前〜現在）

- **やったこと**: 自身のブログ遍歴を口頭で語った。深掘り質問を通じて核心的な洞察を引き出した
- **タイムライン**:
  - 3-4年前: WordPressで読書ブログ。朝5時起き〜6:15、夜9時〜11時、休日は子供の昼寝中に作業。図解はPowerPointで6-7時間かけて作成。サイト設計も素人レベル。「時間をかけてもプロ目線の意見が入っていないので、アウトプットの質が低い」
  - 挫折: 仕事が忙しくなり、朝5時起きも夜作業も難しくなった。進捗も出ない。**完全に諦めた**（フェードアウト、ブログをやめるつもりだった）
  - 2025年9月〜12月: Udemyで生成AI関連動画を視聴。ブログ×生成AIの講師を発見。AIコーチング（GPTsで好き×得意×価値観を3時間深掘り）→サウナに**「腹落ち」**
  - 2025年末〜2026年1月: サウナブログ着手。ラッコキーワード＋Geminiでニーズ調査。Claudeデスクトップアプリで記事作成。キーワードリストはGoogle Spreadsheet
  - 2026年1月: WordPressに落とし込もうとしたが「ポチポチデザインを整えるのにPC張り付き」→子育てと両立できない。Howing1年契約（¥1万+）済み。ドメイン移行は2027年1月まで不可
  - Geminiにディスカッション→Astro発見→YouTube→Cursor→Gemini再相談→最新技術スタック→Cloudflare
  - 現在: 全工程をAI×自然言語で。スマホ×PC連携で外出先からも作業可能
- **感想**: 「戦略の解像度が低い中で我流でずっと続けていた。断片的な知識をもとにやっているので一貫性がなかった。家族にも迷惑をかけながら、将来像が見えない中でやっていた」
- **温度感**: 非常に高い。全ストーリーの根幹
- **関連記事候補**: No.1強化、新記事「共働き×子育て×AIブログ」

### 2026-04-03 / スマホからブログ更新する動機と実体験

- **やったこと**: 記事No.2ブラッシュアップの深掘り。モバイル開発環境を作った動機・利用実態・生活の変化について語った
- **感想（きっかけ）**: 「昼休みに1時間散歩するんですけど、4000-5000歩くらい。YouTube聞きながら歩いてたけど、この時間にClaude Codeに指示を投げられたら生産的じゃないかと。あと出勤前にClaude Code触って承認待ちで止まる。スマホがあれば駐車場でチャチャっと承認できる」
- **感想（利用実態）**: 「記事修正・デザイン修正・戦略相談・定量分析がメイン。新規執筆はPCの方がやりやすい。GSCとGA4をMCPでつないでいるので、スマホから定量分析もできる。ガッツリやるときはPC、サクッとやれる作業はスマホ」
- **感想（変化）**: 「子どもたちと出かけているときも、妻が運転してくれているときに『5分触っていい？』でサクッと修正できる。隙間時間でちょこちょこ進められるようになった」
- **感想（衝撃）**: 「本当にPCのターミナルみたいな感じで、パソコン画面がスマホに映っているみたいな感覚。こんな時代が来たかと衝撃だった」
- **感想（リモートコントロールとの比較）**: 「Claude Codeのリモートコントロールはワンセッション限定で通信が切れたりする。今の環境は安定していて複数セッション立ち上げられる。こっちの方がいい」
- **感想（限界）**: 「唯一の欠点が音声入力と日本語入力ができないこと。直で音声入力する方がだいぶ楽なので、ガッツリやるときはPC」
- **温度感**: 高い。日常の運用に根差した実体験
- **関連記事候補**: No.2ブラッシュアップ（反映済み）
